# 长期记忆

AstrBot 的长期记忆会从已完成的对话中写入用户事实、人物画像和事件摘要，并在后续请求中检索相关内容。该能力位于 WebUI 的 **Alkaid 记忆 / 长期记忆** 页面，目前仍属于实验性功能。

> [!WARNING]
> 当前源码会在运行时无条件初始化长期记忆，并在本地 Agent 对话完成后排队写回；WebUI 和配置档中暂时没有按用户或按配置档关闭的开关。处理敏感、受监管或需要明确同意的数据前，请先评估这一行为以及数据库、备份和日志的保留策略。

## 当前会保存什么

长期记忆包含几类独立记录：

- **事实（facts）**：当前提取器使用明确的中英文规则识别姓名、喜欢和不喜欢等陈述，而不是把整段对话交给 LLM 自由提取。重复事实会合并并更新证据和置信度。
- **人物画像（profiles）**：把同一用户在当前隔离会话中的有效事实整理为简短文本。写入新事实后会自动刷新，也可以在 WebUI 手动刷新。
- **事件摘要（episodes）**：对达到最小长度的已完成轮次保存简短标题和摘要，其中包含截断后的用户文本、助手文本及提取出的事实。
- **操作记录（operation logs）**：记录创建、合并、更新、软删除、恢复和画像刷新等操作及操作者、原因和部分载荷。

这些记录位于 AstrBot 的主 SQLite 数据库中，默认是运行根目录下的 `data/data_v4.db`。修改 `ASTRBOT_ROOT` 会改变运行根目录。备份该数据库时也会备份长期记忆。

## 如何注入模型上下文

每次本地 Agent 请求前，AstrBot 会按当前用户和消息会话检索：

- 一份人物画像；
- 最多 3 条与本次请求相关的事实；
- 最多 1 条相关事件摘要。

它们以临时 `<memory_context>` 内容加入本次请求，并附带“只作为内部上下文、不要逐字引用”的指令。临时注入内容不会作为普通对话消息写回历史，但会发送给当前聊天 Provider。

当 Persona 允许全部工具时，Agent 还会自动获得：

- `search_memory`：搜索当前用户的事实；
- `get_person_profile`：读取当前画像；
- `query_episode`：搜索事件摘要；
- `maintain_memory`：预览、软删除或恢复事实。

Persona 使用明确工具白名单时，这些工具不会自动加入；需要使用时应在 Persona 中显式选择。即使没有工具，自动检索和写回仍会运行。

## 用户与作用域

`person_id` 优先使用平台适配器提供的发送者 ID；缺失时退回消息会话标识。`chat_id` 使用统一消息来源（UMO），其中包含平台、消息类型和会话路由信息。

默认作用域是 `isolated:<chat_id>`：事实、画像和事件只在同一消息会话中检索。代码支持显式的跨作用域共享策略，但当前长期记忆页面没有管理这类策略的入口；普通部署不应假设记忆会跨群、跨私聊或跨平台共享。

## 在 WebUI 中审计

长期记忆页面当前可以：

- 按 Person ID、Chat ID、状态或文本筛选事实；
- 查看事实详情、来源标识、置信度和对应操作记录；
- 软删除或恢复事实，并填写操作原因；
- 查看人物画像并提交刷新任务；
- 查看最近的操作记录和写回队列状态。

页面目前只显示事件摘要数量，不提供事件摘要逐条管理；也不提供通用的事实创建、编辑或永久删除按钮。

## 删除与隐私边界

WebUI 中的“删除”是**软删除**：事实状态从 `active` 改为 `deleted`，正常检索会忽略它，但记录本身仍在数据库中，并且可以恢复。

软删除事实不会自动删除：

- 已生成的人物画像；
- 已生成的事件摘要；
- 操作日志及其中保存的审计载荷；
- 已有备份中的副本。

删除事实后可手动刷新画像；当前实现只有在仍存在有效事实时才会重建画像，因此当最后一条事实被删除时，旧画像也不会被自动硬删除。不要把 WebUI 的软删除视为数据擦除或“被遗忘权”实现。

如果部署需要永久删除、用户退出或保留期限，运维方必须在启用前设计数据库级清理、事件摘要和画像联动、操作日志最小化以及备份过期流程。执行数据库级操作前应停止 AstrBot、制作可验证备份，并在隔离副本上验证清理范围。

## 使用建议

- 告知用户会保存哪些对话衍生数据，并按所在地要求取得同意。
- 不要在对话中发送密码、API Key、身份证件或其他不应长期保存的秘密。
- 定期检查事实、画像、事件数量和操作日志，而不是只看 Agent 回复是否正确。
- 事实提取和检索目前是轻量规则与文本匹配，可能遗漏、误判或引用过时信息；重要决策不能只依赖长期记忆。
- [Persona](./persona) 定义角色和权限；长期记忆按用户与消息会话存储，两者不是同一个数据集合。
